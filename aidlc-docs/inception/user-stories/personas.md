# ペルソナ定義

## 概要

本ドキュメントは、Hono × Inertia.js × React ブログサイトにおけるユーザーペルソナを定義します。
要件定義書（requirements.md）の対象ユーザー分析に基づき、2種類のペルソナを設定します。

---

## Persona 1: 未認証訪問者（ゲストユーザー）

| 項目 | 内容 |
|------|------|
| **ペルソナ名** | 田中 健太（Kenta Tanaka） |
| **年齢** | 28歳 |
| **職業** | Webエンジニア（フロントエンド専門） |
| **技術レベル** | 中級者 |

### 背景・文脈

田中健太は、日々の業務でReactを使っているフロントエンドエンジニア。
技術ブログを定期的に読んで学習することを習慣にしている。
新しいフレームワークやアーキテクチャに興味があり、Honoや Inertia.js についての記事を探している。

### ゴール

- 最新の技術記事を手軽に見つけて読みたい
- 記事の詳細を確認して、自分の学習に役立てたい
- 気に入ったサービスがあれば登録してより深く関わりたい

### 課題・フラストレーション

- 読みたい記事がどこにあるか分からない
- 登録前にサービスの質を確認できない
- ページの読み込みが遅いとすぐに離脱してしまう

### 利用シナリオ

1. 検索エンジン経由でブログに到達する
2. 記事一覧を眺めて興味のある記事を見つける
3. 記事詳細を読んでコメントを確認する
4. サービスの質が高いと判断したらユーザー登録を行う

## Persona 2: 認証済みユーザー（一般ユーザー）

| 項目 | 内容 |
|------|------|
| **ペルソナ名** | 佐藤 美咲（Misaki Sato） |
| **年齢** | 32歳 |
| **職業** | フルスタックエンジニア |
| **技術レベル** | 上級者 |

### 背景・文脈

佐藤美咲は、フロントエンド・バックエンド両方の開発経験を持つエンジニア。
自分の技術的な知見をアウトプットするためにブログを活用したいと考えている。
HonoとCloudflare Workersを組み合わせたアーキテクチャに興味があり、学習内容を記事としてまとめたい。

### ゴール

- 自分の学習内容や知見を記事として発信したい
- 他のエンジニアと技術的な議論をコメント経由で行いたい
- 過去の記事を振り返って内容を更新・改善したい

### 課題・フラストレーション

- 投稿した記事の管理が煩雑だと継続できない
- コメントを通じた交流が少ないとモチベーションが下がる
- 記事編集の UI が使いにくいと投稿頻度が落ちる

### 利用シナリオ

1. ログインして自分のダッシュボードに到達する
2. 新しい記事を投稿する（タイトル・本文を入力）
3. 既存の記事を確認し、必要に応じて編集する
4. 他の記事のコメントを読んで自分もコメントを投稿する
5. 作業が終わったらログアウトする
