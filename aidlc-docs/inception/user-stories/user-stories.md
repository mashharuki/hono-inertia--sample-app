# ユーザーストーリー

## 概要

本ドキュメントは、Hono × Inertia.js × React ブログサイトのユーザーストーリーを定義します。
要件定義書（requirements.md）に記載された機能要件 FR-001〜FR-009 に対応する 8 件のストーリーを管理します。

- **対象ユーザー**: 未認証訪問者 / 認証済みユーザー
- **優先度方式**: MoSCoW（Must / Should / Could / Won't）
- **受け入れ条件**: 各ストーリーに 1〜2 件

---

## ユーザーストーリー一覧

### US-001: 記事一覧の閲覧

| 項目 | 内容 |
|------|------|
| **ID** | US-001 |
| **対応要件** | FR-001 |
| **ペルソナ** | 未認証訪問者（田中健太） |
| **優先度** | Must |

**ストーリー**

> 未認証の訪問者として、公開されている記事の一覧を閲覧したい。
> なぜなら、興味のある記事を見つけて詳細を読むための入口が必要だから。

**受け入れ条件**

1. トップページに記事のタイトルと投稿日時の一覧が表示される
2. 認証なしでアクセスでき、記事一覧が正常に表示される

### US-002: 記事詳細の閲覧

| 項目 | 内容 |
|------|------|
| **ID** | US-002 |
| **対応要件** | FR-002 |
| **ペルソナ** | 未認証訪問者（田中健太） |
| **優先度** | Must |

**ストーリー**

> 未認証の訪問者として、記事の詳細内容とコメントを閲覧したい。
> なぜなら、記事の本文を読み、他の読者の反応を確認したいから。

1. 記事タイトル・本文・投稿日時・コメント一覧が表示される
2. 認証なしでアクセスでき、記事詳細が正常に表示される

### US-003: ユーザー登録

| 項目 | 内容 |
|------|------|
| **ID** | US-003 |
| **対応要件** | FR-005 |
| **ペルソナ** | 未認証訪問者（田中健太） |
| **優先度** | Must |

**ストーリー**

> 未認証の訪問者として、メールアドレスとパスワードで新規アカウントを作成したい。
> なぜなら、記事の投稿やコメントといった認証必須の機能を利用したいから。

**受け入れ条件**

1. メールアドレスとパスワードを入力してアカウント登録できる
2. 登録成功後、自動的にログイン状態になり認証済みユーザーの機能が利用できる

### US-004: ログイン

| 項目 | 内容 |
|------|------|
| **ID** | US-004 |
| **対応要件** | FR-006 |
| **ペルソナ** | 未認証訪問者（田中健太） |
| **優先度** | Must |

**ストーリー**

> 既存アカウントを持つ訪問者として、メールアドレスとパスワードでログインしたい。
> なぜなら、記事投稿やコメント投稿などの認証が必要な機能を使いたいから。

**受け入れ条件**

1. 正しいメールアドレスとパスワードを入力するとログインに成功する
2. 誤った認証情報を入力した場合はエラーメッセージが表示される

### US-005: 記事の投稿

| 項目 | 内容 |
|------|------|
| **ID** | US-005 |
| **対応要件** | FR-003 |
| **ペルソナ** | 認証済みユーザー（佐藤美咲） |
| **優先度** | Must |

**ストーリー**

> 認証済みユーザーとして、新しい記事をタイトルと本文を入力して投稿したい。
> なぜなら、自分の技術的な知見を他のエンジニアと共有したいから。

**受け入れ条件**

1. タイトルと本文を入力して投稿すると記事が公開される
2. 投稿した記事は記事一覧と記事詳細ページで確認できる

---

### US-006: 記事の編集

| 項目 | 内容 |
|------|------|
| **ID** | US-006 |
| **対応要件** | FR-004 |
| **ペルソナ** | 認証済みユーザー（佐藤美咲） |
| **優先度** | Should |

**ストーリー**

> 認証済みユーザーとして、自分が投稿した記事のタイトルと本文を編集したい。
> なぜなら、誤字の修正や内容の更新・改善を行いたいから。

**受け入れ条件**

1. 自分の記事の編集フォームにアクセスし、タイトル・本文を変更して保存できる
2. 編集後の変更内容が記事詳細ページに即時反映される

### US-007: ログアウト

| 項目 | 内容 |
|------|------|
| **ID** | US-007 |
| **対応要件** | FR-007 |
| **ペルソナ** | 認証済みユーザー（佐藤美咲） |
| **優先度** | Must |

**ストーリー**

> 認証済みユーザーとして、明示的にログアウト操作を行いたい。
> なぜなら、共有端末などで使用した際にセッションを安全に終了させたいから。

**受け入れ条件**

1. ログアウト操作を行うとセッションが無効化され、未認証状態に遷移する
2. ログアウト後に認証必須ページへアクセスしようとするとログインページへリダイレクトされる
